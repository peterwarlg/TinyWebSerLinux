1. static

全局/静态储存区

  • .data 初始化的全局储存区
  • .BSS 未初始化的全局储存区

static作用

  1. 隐藏
    1. 所有static修饰的变量和函数，只对该文件可见，别的文件哪怕extern也不可见
  1. 保持变量内容的持久
    1. 定义在函数内部的static变量，不会随函数生命周期消失；只会被初始化一次；
    1. 储存在静态数据区，只会在程序刚开始运行时就初始化且只初始化一次
    1. 全局变量和静态变量储存在静态储存区，static控制可见范围
  1. 默认初始化为0
  1. 类成员声明为static
    1. 模块内的static变量可以被模块内所有函数访问，模块外函数不可访问
    1. 模块内的static函数只被模块内所用函数访问，使用范围被限制在该模块
    1. static修饰的成员对象必须在类外初始化，因为其先于对象而存在
    1. static成员没有this指针，所以不能使用其他非static成员
    1. 不能被virtual修饰，因为其没有this指针，虚函数的调用在于使用this指向的vptr的vtable

2. const

  1. 不考虑类的情况
    • 定义时必须初始化，初始化之后无法更改
    • const形参可以接收非const实参
  1. 考虑类的情况
    • const成员变量
      • 不能在类中定义时初始化，因为不同对象可能有不同值
      • 只能通过构造函数初始化列表初始化，必须有构造函数

const修饰函数参数

  • const形参可以接收非const实参
  • 对于非内部数据类型的输入参数，应该将“值传递”的方式改为“const引用传递”，目的是提高效率。例如将void Func(A a) 改为void Func(const A &a)。
  • 对于内部数据类型的输入参数，不要将“值传递”的方式改为“const引用传递”。否则既达不到提高效率的目的，又降低了函数的可理解性。例如void Func(int x) 不应该改为void Func(const int &x)。

const修饰函数返回值

该返回值只能被赋值给const修饰的同类型变量

  • 当返回一个const修饰的指针时，接收方应该也是const修饰的指针
  • 如果函数返回值采用“值传递方式”，由于函数会把返回值复制到外部临时的存储单元中，加const 修饰没有任何价值。

const修饰成员函数 | const加在函数名之后

  • 表明该函数不会修改成员变量
  • 只能调用const修饰的成员函数

const对象

  • const对象只能访问const函数
  • const对象不能修改成员变量 | 可以修改成员指针指向的对象
  • 加上mutable的成员变量可以被任何函数和对象直接修改

extern

定义

置于变量或函数之前，标识变量或函数定义在别的文件中，提示编译器遇到此变量和函数时在其他模块中寻找其定义。此外extern也可用来进行链接指定。

第一个作用

第二个作用：修饰函数和变量

  1. 此变量或函数在别处定义，要在此处引用
  1. 告诉编译器有一个type类型的变量x在某处存在，而编译器无需知道在哪，当所有文件编译完成后，由链接器将所有的x连接到某一处定义

修饰函数

事实证明，当一个函数被声明或定义时，外部关键字是隐含假设的。当我们写作的时候

int foo(int arg1, char arg2);

编译器将其视为：

extern int foo(int arg1, char arg2);

修饰变量：代表只声明不定义

变量声明时不分配内存；

与int i不同，extern int i 代表只声明变量，不做定义，此时i不能被使用

extern int var;
int main(void)
{
  var = 10;
  return 0;
}

try 、catch、throw

函数 A 在执行过程中发现异常时可以不加处理，而只是“拋出一个异常”给 A 的调用者，假定为函数 B。

拋出异常而不加处理会导致函数 A 立即中止，在这种情况下，函数 B 可以选择捕获 A 拋出的异常进行处理，也可以选择置之不理。如果置之不理，这个异常就会被拋给 B 的调用者，以此类推。

如果一层层的函数都不处理异常，异常最终会被拋给最外层的 main 函数。main 函数应该处理异常。如果main函数也不处理异常，那么程序就会立即异常地中止。

catch和throw的都是一个对象

try {
    语句组
}
catch(异常类型) {
    异常处理代码
}
...
catch(异常类型) {
    异常处理代码
}

catch所有类型

catch(...) {
    ...
}

例子

#include <iostream>
using namespace std;
int main()
{
    double m ,n;
    cin >> m >> n;
    try {
        cout << "before dividing." << endl;
        if( n == 0)
            throw -1; //抛出int类型异常
        else
            cout << m / n << endl;
        cout << "after dividing." << endl;
    }
    catch(double d) {
        cout << "catch(double) " << d <<  endl;
    }
    catch(int e) {
        cout << "catch(int) " << e << endl;
    }
    cout << "finished" << endl;
    return 0;
}

函数自身throw并且catch

int CountTax(int salary)
{
    try {
        if (salary < 0)
            throw - 1;
        cout << "counting tax" << endl;
    }
    catch (int) {
        cout << "salary < 0" << endl;
    }
    cout << "tax counted" << endl;
    return salary * 0.15;
}

异常的再抛出：自身不处理 并throw给上一级

double Devide(double x, double y)
{
    if (y == 0)
        throw string("devided by zero");
    cout << "in Devide" << endl;
    return x / y;
}
int main()
{
    double f = 1.2;
    try {
        f = Devide(3, 0);
        cout << "end of try block" << endl;
    }
    catch (string e) {
        cout << e << endl;
    }
    cout << "finished" << endl;
    return 0;
}

如果拋出的异常是派生类的对象，而 catch 块的异常类型是基类，那么这两者也能够匹配，因为派生类对象也是基类对象。

函数捕获异常并处理但还是希望通知上级函数

catch (string s ) {
        cout << "CountTax error : " << s << endl;
        throw; //继续抛出捕获的异常
    }

异常声明列表

  • 为了增强程序的可读性和可维护性，使程序员在使用一个函数时就能看出这个函数可能会拋出哪些异常，C++ 允许在函数声明和定义时，加上它所能拋出的异常的列表，具体写法如下：
  • void func() throw (int, double, A, B, C); 或者void func() throw (int, double, A, B, C){...}
  • 如果：void fun() throw() {} 表示不抛出异常
  • 函数如果拋出了其异常声明列表中没有的异常，在编译时不会引发错误，但在运行时， Dev C++ 编译出来的程序会出错；用 Visual Studio 2010 编译出来的程序则不会出错，异常声明列表不起实际作用。
